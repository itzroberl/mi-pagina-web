## Quick bounty triage sample

Before commenting `/attempt`, check:

1. The bounty amount is explicit.
2. The issue is still open.
3. There are not many recent PRs already solving it.
4. The maintainer has accepted similar PRs before.
5. The task can be validated locally without private credentials.
6. The expected change fits in a small PR.
7. The issue does not ask for unsafe behavior, private prompts, secrets, scraping credentials, or out-of-scope testing.

## Mini `/attempt` comment sample

```text
/attempt #ISSUE_NUMBER

I can take this. My plan:
- reproduce the current behavior
- make the smallest scoped change
- add or update the focused test/docs
- include validation commands and screenshots if relevant

I will avoid unrelated refactors.
```
